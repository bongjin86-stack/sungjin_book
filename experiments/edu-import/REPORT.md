# HWP Import 가능성 조사 — 결과 보고

> 작성: 2026-05-20
> 격리 폴더: `experiments/edu-import/` (본앱 단절)
> 대상: 평가원 양식 .hwp 5개 (국어/영어/수학/사탐/과탐)
> 결론 한 줄: **pyhwp `hwp5txt`/`hwp5html`로 텍스트 추출은 충분히 가능. 다만 객관식 보기는 과목별로 보존율이 다르고, 표·도형·수식은 별도 처리 경로가 필요하다.**

## 1. 테스트한 파일과 도구

| 파일 | 크기 | hwp5txt 줄수 | hwp5txt 결과 |
|---|---:|---:|---|
| 평가원 과탐 양식.hwp | 481 KB | 277 | 추출 OK, 보기는 표 안에 묻힘 |
| 평가원 사탐 양식.hwp | 536 KB | 212 | 추출 OK, 보기는 표 안에 묻힘 |
| 평가원 수학 양식.hwp | 770 KB | 438 | 추출 OK, 보기 본문이 도형/수식이라 빠짐 |
| 평가원 영어 양식.hwp | 3.5 MB | 403 | 추출 OK, **보기 ①~⑤ 본문 보존** |

도구:
- `hwp.js` (npm, Apache-2.0, v0.0.3) — **부적합**. 평가원 양식의 알 수 없는 컨트롤에서 `Except: control, Recived: undefined` throw. 신뢰성 낮음.
- `pyhwp` (PyPI `pyhwp` 0.1b15, BSD/GPL 듀얼) — **적합**. 5개 파일 전부 추출 성공. `olefile`·`lxml` 의존. 경고는 `UnderlineStyle 15` 미정의 하나뿐, 추출 자체엔 영향 없음.

설치 방법(격리):
```
cd experiments/edu-import
python -m venv .venv
.venv/Scripts/pip install pyhwp six
```
루트 `package.json`·루트 venv 일절 미오염.

## 2. 추출 품질 평가

`grep -c` 기준 보기 글리프(①②③④⑤) 출현 횟수와 표/그림 placeholder 분포:

| 파일 | ①~⑤ | `<표>` | `<그림>` | 문항번호(`^N\.`) |
|---|---:|---:|---:|---:|
| 과탐 | 102 | 60 | 10 | 20 |
| 사탐 | 73 | 51 | 4 | 0 (※ `N.` 패턴 안 씀) |
| 수학 | 134 | 20 | 55 | 46 |
| 국어 | 586 | 44 | 52 | 56 |
| 영어 | 241 | 23 | 16 | 45 |

### 살아남는 것
- ✅ **발문/지시문 텍스트**: "다음은 ~에 대한 자료이다.", "옳은 설명만을 <보기>에서 고른 것은?" 등 거의 완벽.
- ✅ **문항 번호**: `1.`, `2.` 형태로 단락 머리에 보존. 사탐은 번호가 도형 안에 있어 텍스트로 빠짐.
- ✅ **배점**: `[3점]` 텍스트 그대로 살아남.
- ✅ **국어/영어의 객관식 보기**: ①~⑤ 글리프 + 그 뒤 텍스트가 한 줄로 연결됨. 예: `①밑줄 긋기는 일반적인 독서 상황에서 도움이 된다.`
- ✅ **`<보기>` 박스 마커**: 평가원 표준 `<보기>` 표기 보존.

### 손실되는 것 (hwp5txt 단순 추출 기준)
- ❌ **표 안의 보기/자료**: 사탐·과탐은 객관식 5지선다를 표로 그리는 경우가 많아 `<표>` placeholder로 묻힘. 본문은 표 안 텍스트를 펼치지 않음.
- ❌ **수식·화학식·도식**: 수학·과탐의 핵심. 평가원은 수식을 OLE 객체/도형으로 박아 텍스트 흐름에 안 들어옴. `<그림>`으로만 표시.
- ❌ **수학 보기 본문**: ①②③④⑤가 한 줄에 5개 모인 뒤 옆에 텍스트 없음 — 보기 자체가 수식이라 그림 영역.
- ❌ **이미지/그래프**: 모두 `<그림>` placeholder.

### 복구 가능성 (다음 단계 확인 필요)
- 🟡 **표 내부**: `hwp5html` 변환은 표 안까지 펼쳐서 HTML `<table>`로 뽑힘(이번 실험에서 일부 확인). 인코딩 처리 추가 필요(콘솔 latin-1로 깨졌지만 추출 자체는 됨). XML 덤프(`hwp5proc cat-records`)로도 표 구조 접근 가능.
- 🔴 **수식**: HWP의 수식 객체는 OLE binary. 텍스트로는 복구 불가, MathML 변환은 한컴 자체 도구가 필요.
- 🔴 **이미지**: bindata 폴더에 PNG/EMF로 떨어지지만 위치·캡션 매핑은 별도 작업.

## 3. 자동 문항 구조화 적합성

| 과목 유형 | 적합성 | 비고 |
|---|---|---|
| 국어 (지문+5지선다) | **🟢 매우 적합** | 지문·발문·보기 모두 텍스트로 살아남. JSON 변환 가능. |
| 영어 (지문+5지선다) | **🟢 매우 적합** | 국어와 동일 패턴. |
| 사탐/과탐 (표 자료+5지선다) | **🟡 조건부 적합** | `hwp5html`로 표 펼치면 보기는 살아남. 자료 표는 별도 처리. |
| 수학 (수식 5지선다) | **🔴 부적합** | 보기 자체가 수식. 이미지로 박힌 보기를 수식으로 다시 풀어야 하는데 LLM·OCR 단계 필요 — 1단계 범위 밖. |

**판단**: 1차 MVP는 **국어/영어부터** 가야 한다. 수학·과탐 수식은 별도 트랙(이미지 보기를 그대로 박는 시험지)에서 다룬다.

## 4. 다음 단계 추천 (순서)

1. **`hwp5html`로 표까지 펼친 추출 1회 더** — 평가원 사탐/과탐의 보기가 표 안에 갇혀 있는데, HTML/XML 단계에선 펼쳐진다는 것이 이번에 확인됨. 인코딩 OK한 깨끗한 추출 한 번만 더 해서 5과목 모두 보기 보존율 재측정.
2. **문항 단위 분할 규칙 설계** — 평가원은 문항을 `숫자.` + 빈 줄로 구분하는 패턴이 일관. 정규식 1줄로 분리 가능할 가능성 큼. 국어 텍스트로 PoC.
3. **`web/types/question.ts` 초안** — 지문/발문/보기/배점만 가진 최소 스키마. `experiments/edu-import/` 안에 먼저 작성해서 본앱에 들이기 전 검증.
4. **국어 1회분 → JSON → Typst 조판** — 시험지 템플릿 골격 만들어 PDF 한 장 떨궈본다. 사용자 검수.
5. **수식·표·이미지**는 **명시적 보류** — 1차 MVP에 안 들어감. 진짜 1차가 통과되면 별도 트랙.

## 7. 한국어 문항 분할 PoC 결과 (2026-05-20 추가)

> 범위: 평가원 국어 양식 1개. 수식/이미지/표 복원 제외. 본앱 코드 변경 0건.
> 스크립트: `scripts/split-korean-questions.py`
> 산출물: `out/korean-questions.json` (schema `edu-import/v0`)

### 7.1 성공률 정량

| 지표 | 결과 | 평가 |
|---|---|---|
| 묶음 인식 (`[N~M]`) | **14/14** | 🟢 완벽 |
| 문항 분할 | **56/56** (1~34 공통 + 35~45 선택1 + 35~45 선택2) | 🟢 완벽 |
| 묶음↔문항 부착 (`passage_id`) | **56/56** | 🟢 완벽 |
| 보기 ①~⑤ 5개 보존 | **52/56 (93 %)** | 🟢 우수 |
| 3점 문항 자동 인식 | **12개** (`[3점]` 태그) | 🟢 정확 |

손실 4건은 **분할기 책임이 아니라 추출 단계(hwp5txt) 한계** — 보기 본문이 한컴 표 안에 그려져 텍스트 흐름에 안 나옴. `hwp5html` 또는 XML 단계에서 회수 가능.

### 7.2 실패/애매 사례 3건

**사례 A — 문항 17 (어휘 바꿔쓰기)**
```
stem: "문맥상 ⓐ~ⓔ와 바꿔 쓰기에 가장 적절한 것은?"
choices: 3개만 (①②③) — ④⑤는 표 안에 묻힘
```
원인: 보기 본문이 5칸 표로 그려짐. 일부 셀만 텍스트 흐름에 빠져나옴. 분할기는 글리프 순서 깨짐(`['②','④','⑤']` 같은)을 경고로 잡아냄.

**사례 B — 문항 20 (`<학습 활동>` 수행 결과)**
```
stem: "<학습 활동>을 수행한 결과로 적절하지 않은 것은? <표> <표>"
choices: 0개
```
원인: 학습활동 자료 표 + 보기 표가 연속으로 등장. 모두 `<표>` placeholder로 묶임. 발문 자체는 살아남(분할기는 placeholder를 발문에 합침).

**사례 C — 문항 39 (선택1, 발문 중간에 그림)**
```
stem: "다음은 '학생3'이 <그림> 을 준비하면서 (가)의 대화 내용을 참고한 메모이다. ..."
choices: 0개 (보기는 표 안)
```
원인: 발문이 줄바꿈으로 끊겨 있고 중간에 `<그림>`이 끼어듦. **분할기는 멀티-라인 발문을 잘 합쳐 복원함 ✅.** 보기 손실만 별개 문제.

### 7.3 분할기 작동 패턴 요약

| 단서 | 정규식 | 동작 |
|---|---|---|
| 묶음 헤더 | `^\[(\d+)\s*[~～]\s*(\d+)\]` | 새 passage 시작, 이전 문항 flush |
| 문항 헤더 | `^(\d{1,3})\.` | 새 question, 보기/문항 flush |
| 보기 헤더 | `^([①-⑤])` | 새 choice, 이전 choice flush |
| 배점 | `\[(\d+)\s*점\]` | 발문에서 추출 후 제거 |
| placeholder | `<(그림\|표)>` | 발문/보기에 끼어들면 합치고 플래그 |

상태 머신은 빈 줄을 단지 토큰 경계로만 쓰고 강제 종료 신호로 안 씀 → 발문/보기가 빈 줄로 나뉘어도 안전.

### 7.4 사업 검증 함의

- **국어 56문항이 완전 분할 가능** → "국어/영어 평가원형 HWP → 문항 JSON → 시험지 PDF" 1차 MVP의 **앞 절반이 입증됨**.
- 보기 텍스트 손실 4건은 `hwp5html` 1회 추가 패스로 회수 시도 가치 있음 (선택 작업).
- 발문에 끼어든 그림/표 placeholder 26건은 시험지 PDF에서 **빈 박스(`[그림]`/`[표]`)로 표시**하면 사람 검수에서 그림 붙이는 단계로 자연 연결.
