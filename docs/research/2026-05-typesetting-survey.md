## 1. 한 줄 요약

- 글로벌 1, 2위(**Vellum / Atticus**)는 **"테마 선택 + 트림 사이즈 노출"이 기본축**이고, 폰트·자간·들여쓰기·H레벨은 **테마 안에서만** 만진다.
- **자간(letter-spacing)을 사용자에게 노출하는 도구는 11개 중 0개**다. Affinity Publisher(전문가용 DTP) 한 곳만 예외.
- **여백 4방향**도 거의 노출 안 한다. 노출되는 곳도 보통 **inside / outside 2축**(KDP·IngramSpark·Vellum 모두 동일).
- POD 풀패키지(KDP·Ingram·Lulu·부크크·퍼플)는 **조판을 안 해준다**. 사용자가 PDF를 만들어 올린다.
- 결론: 우리 ③번 결정("판형 + 스타일 2개")은 **시장 평균보다 약간 적은 수준이지만 Vellum보다 많이 적지는 않다**. C/D 섹션 참고.

## 3. 도구별 상세

### 3.1 Vellum (Mac, 시장 1위)

- **포지션**: "Choose your trim size, and Vellum does the rest." 자동화 1순위.
- **노출 옵션 (Print Settings)**: 트림 사이즈, **inside/outside 마진(2축만)**, 폰트 크기 슬라이더(포인트 비공개), 줄간격 슬라이더, 챕터 시작(첫장만 vs 매장 우측), 페이지 번호 시작 지점, 이미지 흑백/컬러, 블리드(필요시/항상).
- **숨겨진 것**: top/bottom 마진, 자간, 한자, 표, H2~H6, 각주(엔드노트만 지원).
- **테마(Style)**: 24~26종. 스타일별 변형 포함하면 **82개**. 본문 폰트 26종 한정(시스템 폰트 추가 안 됨).
- **데이터 모델 노출**: 🚫 (Vellum 전용 `.vellum` 파일).
- **차별화**: UI 깔끔함·자동 위도우/오펀 처리·실시간 미리보기 1위.
- **가격**: 일회성 $249.99 (인쇄+ebook). Mac 전용.
- **블리드/CMYK/면배치**: 블리드 자동, CMYK 미노출, 면배치 자동.

### 3.2 Atticus (Web/PWA, Vellum 대안)

- **포지션**: Vellum + 비문학 + 윈도/크롬북 호환. 가장 "옵션 많은" 도구.
- **노출 옵션 (Custom Theme + Print Layout)**: 트림 사이즈, inside/outside 마진, 들여쓰기 깊이, 양끝/하이픈 토글, 폰트(본문 15종+헤딩 Google Fonts 1500+), 폰트 크기, 줄간격, H2~H6 헤딩, 드롭캡, 페이지 번호 위치, 헤더(좌/우 페이지 별도), 풋노트·엔드노트, 콜아웃 박스, 씬 브레이크.
- **숨겨진 것**: 자간, 한자/루비, top/bottom 마진(일부 미노출), 표(제한적).
- **테마**: 17종 프리셋 + 커스텀 빌더.
- **데이터 모델 노출**: 🚫 (Atticus 클라우드).
- **차별화**: 비문학 대응(콜아웃·H6), 폰트 자유도, 가격 절반.
- **가격**: 일회성 $147 (평생).
- **블리드/CMYK/면배치**: 블리드 자동, CMYK 미노출, 면배치 자동.

### 3.3 Reedsy Studio / Reedsy Book Editor (Web, 무료)

- **포지션**: "양식은 우리가, 작가는 글만." 가장 적은 노출.
- **노출 옵션**: 트림 사이즈 4종(6×9, 5.5×8.5, 5×8, 4.25×6.87), 폰트 크기 small/medium/large, 폰트 테마 3종(Reedsy/Classic/Romance), 챕터 제목, 엔드노트→풋노트 변환 옵션.
- **숨겨진 것**: 마진(자동), 줄간격, 자간, 들여쓰기 세부, 정렬, H레벨 세부, 페이지 번호 위치, 러닝 헤드.
- **테마**: 3종.
- **데이터 모델**: 🚫.
- **차별화**: 100% 무료, "마진·간격·들여쓰기·과부 자동". 우리 정책과 가장 비슷.
- **가격**: 무료 (Studio Essential $4.99/월은 별개 기능).
- **블리드/CMYK/면배치**: 자동.

### 3.4 Scrivener (참고용, 작가용 도구)

- **포지션**: 조판 도구 아님. **장(章) 단위 글쓰기 + Compile로 PDF/EPUB/워드 변환.**
- **노출 옵션 (Compile 다이얼로그)**: 사실상 거의 모두. 폰트, 사이즈, 마진, 들여쓰기, 정렬, H레벨, 인용문, 이미지, 표, 풋노트.
- **차별화**: 글쓰기 환경(코르크보드·시놉시스), 조판은 부차.
- **데이터 모델**: 🚫.
- **가격**: 일회성 $59.99 (Mac/Win), $23.99 (iOS).
- **결론**: 우리 경쟁자 아님. 작가가 Scrivener에서 쓰고 → Vellum/Atticus/Reedsy/우리에게 와서 조판한다.

### 3.5 Affinity Publisher 2 (DTP 표준)

- **포지션**: InDesign 대안. 전문 디자이너용. 사용자가 모두 만진다.
- **노출 옵션**: 모든 것. 자간(letter-spacing), 베이스라인 그리드, 마스터 페이지(홀/짝/첫쪽 분기), CMYK, PMS 별색.
- **차별화**: 한 번 사면 평생, $99.99 (3앱 묶음). 2025년 10월 단종 → "Affinity by Canva" 프리미엄 통합.
- **결론**: 우리 경쟁자 아님. **참고 기준**(자유도 100% 노출이 어떻게 생겼는지).

### 3.6 Amazon KDP (글로벌 1위 셀프출판)

- **포지션**: 유통/POD 플랫폼. **조판 안 해준다.** 사용자가 PDF 업로드.
- **노출**: 트림 사이즈(custom 4"×6" ~ 8.5"×11.69"), 마진 4방향(가이드만), 블리드 0.125"(가이드만).
- **부속 도구**: Kindle Create(무료, 단순). 표·풋노트·블리드 이미지 미지원.
- **데이터 모델**: 🚫. 사용자 PDF/Word를 받음.
- **가격**: 무료 + POD 마진(60%까지 인세, 7×9 표준 기준).
- **블리드/CMYK/면배치**: 사용자 책임. CMYK 변환 자동(흑백 권장).

### 3.7 IngramSpark

- **포지션**: 서점 유통(반즈앤노블·바보스 등) 강점. 조판 안 해준다.
- **노출**: 트림 사이즈(다수), 마진 최소 0.5"(13mm), 바인드측 화이트 스트립 0.125", 블리드 0.125", 24pt 미만 텍스트 K100%(검정 단색) 강제.
- **부속 도구**: 표지 템플릿 생성기(이메일 발송).
- **가격**: 무료 (2022년 setup fee 폐지). POD 마진 사용자 결정.

### 3.8 Lulu

- **포지션**: 가장 오래된 POD. 디자이너 친화. 조판은 안 함.
- **노출**: 트림 사이즈(다수), 마진 최소 0.5", 거터 0.2~0.3" (60p 이하 제외).
- **부속 도구**: 무료 표지 디자이너(웹), 본문 템플릿 PDF/DOCX/INDD.
- **데이터 모델**: 🚫.
- **가격**: 무료. 인세: 직판 80%, 전세계 유통 80%(소매 할인 55% 제하고).

### 3.9 Draft2Digital

- **포지션**: ebook 1순위 무료 변환 + 인쇄 자동 레이아웃(2022~).
- **노출 옵션**: 21개 템플릿, 드롭캡, 헤더(그래픽/플로리시), 씬 디바이더, 위도우/오펀 자동.
- **폰트/마진/줄간격**: 템플릿 안에서만 자동(🔧). 사용자 선택지 거의 없음.
- **데이터 모델**: 🚫. Word 업로드 → "Heading 1"만 표시하면 자동 변환.
- **가격**: **완전 무료**(D2D 유통 안 써도 무료). EPUB·MOBI·PDF 모두.
- **차별화**: 단일 워드 → ebook + 인쇄 동시 일관성.

### 3.10 부크크 (한국 1위)

- **포지션**: 한국 자가출판 표준. 조판 안 해준다.
- **노출**: 판형(A5, B5, 신국판 등), 종이(일반/고급), 흑백/컬러, 무선/양장 제본. 폰트는 KoPub 권장(직접 박아 PDF로 올림).
- **데이터 모델**: 🚫. PDF 업로드 (한글/워드는 PDF 변환 후 업로드).
- **부속**: 워드용 본문 템플릿 제공(파일 다운로드, 자동 조판 아님).
- **가격**: 무료 가입. 인세 35%(부크크 직판) / 15%(외부 유통). 정가 사용자 결정. 최소 50p.
- **블리드/CMYK/면배치**: 사용자 책임. 자동 규격 체크 도구만 제공.
- **차별화 1줄**: 한국어 시장의 "etc" 흡수, ISBN 무료, 영풍문고/교보 등 외부 유통 연동.

### 3.11 퍼플 (PubPle, 교보문고)

- **포지션**: 교보문고 직영 POD. 조판 안 해준다.
- **노출**: 판형(국판/신국판/A4/B5), 표지 재질(아르떼 210g 등), 본문 내지(스노우 100g 등), 무선/날개 제본.
- **데이터 모델**: 🚫. PDF 업로드.
- **가격**: 무료. **인세 정가 20%(개인) / 30%(대활자본).** 평균 출고 3일.
- **차별화 1줄**: 교보문고 매대 노출.

## 4. 섹션 A — 공통 패턴 (11도구 횡단 결론)

| 항목 | "노출"이 표준인가? | 근거 |
|---|---|---|
| 트림 사이즈 | **노출이 표준** | 11/11 모두 노출 |
| 폰트 선택 | **숨김이 표준** (테마 소속) | Vellum 26종 한정, Reedsy 3테마 한정, KDP·POD 5사는 사용자 PDF에 박혀서 옴 |
| 폰트 크기 | **숨김이 다수** | Vellum/Atticus만 슬라이더, Reedsy는 small/med/large |
| 줄간격 | **숨김이 다수** | 같은 양상. 테마/슬라이더로만. |
| **자간** | **노출 0%** | Affinity(전문가 DTP) 1곳만 노출. 작가용 도구 0/10. |
| 여백 4방향 | **2축(inside/outside)이 표준** | Vellum/KDP/Ingram/Lulu 모두 inside·outside 위주. top/bottom은 거의 자동. |
| 들여쓰기 | **숨김이 표준** | Atticus만 깊이 조절. |
| 양끝 정렬 | **숨김 + 양끝 기본** | KLREQ도 양끝이 한국어 기본. |
| 챕터 우측 시작 | **테마가 결정** | Vellum/Atticus만 토글, Reedsy 자동. |
| H2~H6 | **양극** | Atticus만 비문학용 H6, 나머지는 H1~H3. |
| 자간/한자/루비 | **0%** | 어느 도구도 노출 안 함. |
| 블리드 | **자동이 표준** | 작가용 도구 모두 자동, POD는 가이드 제공. |
| **CMYK·면배치** | **숨김 100%** | Affinity 1곳만 노출. |

**한 줄 결론(한국어):**
> **"자간·여백 4방향·면배치를 사용자에게 노출하는 게 시장 표준이 아니다."** 작가용 도구(Vellum/Atticus/Reedsy/D2D)는 **테마 + 트림 사이즈**가 사실상 노출 한도이고, 폰트/줄간격/들여쓰기는 **테마 안에서 자동** 또는 **슬라이더 한두 개**로만 만진다.

## 5. 섹션 B — 우리 ③번 결정 평가

**우리 결정**: 노출은 "판형 + 스타일 2개"만. 자간·여백·폰트는 노출 안 함.

| 비교 대상 | 우리보다 더 노출 | 우리와 같음/더 적음 |
|---|---|---|
| Vellum | + 폰트 크기, 줄간격 (슬라이더 2개) | 같음: 자간·top/bottom 마진·폰트 직접선택 모두 숨김 |
| Atticus | + 폰트 크기, 줄간격, 들여쓰기 깊이, H2~H6, 헤더 위치, 풋노트, 1500+ 헤딩 폰트 | 같음: 자간 숨김 |
| Reedsy | + 폰트 크기 small/med/large | 같음/적음: 마진·정렬·헤더 모두 자동 |
| Draft2Digital | 같음 (테마 안에서만 자동) | 같음 |
| 부크크/퍼플/KDP/Ingram/Lulu | 우리와 다른 카테고리 (조판 안 함) | — |

### 평가: **(ii) 적당하다 — 다만 1단계용으로**

- **시장 평균보다 약간 적은 수준이지만 Reedsy/D2D와 비슷, Vellum과도 큰 차이 없음.**
- **Atticus 대비 적게 노출**되는 게 사실. Atticus 사용자 후기에서 "옵션이 너무 많아 막막했다"는 평이 다수 → **장점이지 단점 아님**.
- **위험 신호**: 최소 폰트 크기 슬라이더 1개와 줄간격 슬라이더 1개는 **시장 표준**. Vellum도 이 두 개는 노출함. 한국 작가 시각용("내 책 글씨 너무 작아요" 호소 대응) 대비 **폰트 크기 -1/0/+1 정도의 약한 노출**은 2단계에서 검토.
- **권장 단계 전략**:
  - **1단계 (현재)**: 판형 + 스타일 2개. 그대로 유지.
  - **2단계 후보 (B+)**: + "폰트 크기 약 슬라이더 (-1/0/+1)" + "들여쓰기 on/off". 둘 다 KLREQ 호환.
  - **3단계 후보 (B++)**: + "챕터 시작 우측 강제 토글" + "각주 vs 미주 토글".
  - **하지 말 것**: 자간 노출, 여백 4방향 노출, 폰트 직접 선택 노출. 시장 0%.

## 6. 섹션 C — JSON 스키마 v2 윤곽 (Atticus/Vellum 베끼기)

**전제**: 우리 콘텐츠 JSON은 **렌더러 중립**(Typst/InDesign/HTML 다 받게). Atticus/Vellum은 자체 바이너리지만 그들의 **요소 분류**가 사실상 표준임.

```jsonc
{
  "schema_version": 2,
  "book": {
    "title": "태평천하",
    "subtitle": "한국 풍자문학의 정수",
    "authors": ["채만식"],
    "language": "ko-KR",
    "isbn": null
  },
  "format": {
    // 사용자 노출 1: 판형
    "trim_id": "shinkukpan_152x225",   // 신국판
    // 사용자 노출 2: 스타일 2개
    "theme_id": "classic",              // classic | modern (1단계)
    "separator_id": "ornament_v1"       // 챕터 사이 장식
  },
  "front_matter": [
    { "type": "half_title" },
    { "type": "title_page" },
    { "type": "copyright", "body": "..." },
    { "type": "toc", "auto": true },
    { "type": "dedication", "body": "..." },
    { "type": "epigraph", "body": "...", "attribution": "..." }
  ],
  "body": [
    {
      "type": "chapter",
      "id": "ch01",
      "number": 1,                       // 또는 "prologue"
      "title": "윤직원 영감 귀가지도(歸家之圖)",
      "subtitle": null,
      "start_recto": true,               // Atticus/Vellum 표준 — 챕터 우측시작
      "blocks": [
        { "type": "paragraph", "text": "..." },
        { "type": "paragraph", "text": "...", "first_in_section": true },
        { "type": "scene_break" },
        { "type": "blockquote", "text": "...", "attribution": null },
        { "type": "verse", "lines": ["..."] },
        { "type": "heading", "level": 2, "text": "..." },
        { "type": "image", "src": "img/...", "caption": "...", "full_bleed": false },
        { "type": "footnote", "ref": "fn1", "text": "..." },
        { "type": "list", "kind": "ordered", "items": ["..."] },
        { "type": "list", "kind": "unordered", "items": ["..."] },
        { "type": "callout", "kind": "tip", "text": "..." }    // 비문학용 (Atticus 모방)
      ]
    }
  ],
  "back_matter": [
    { "type": "about_author", "body": "..." },
    { "type": "endnotes", "auto": true }
  ],
  "inline": {
    // 인라인 마크업은 paragraph.text 안에 마크다운/HTML 부분집합으로
    // **bold** *italic* `code` ^^smallcaps^^ [漢字]{ruby:한자}  // 한자 루비는 한국 시장 특수 (D 섹션)
    "spec": "subset_markdown_v1"
  },
  "_locked": {
    // 사용자에게 노출하지 않는 항목 (테마/렌더러가 결정)
    "font_body": "Noto Serif CJK KR",
    "font_size": 10,
    "leading": 18,
    "letter_spacing": 0,
    "margin": { "inside": 20, "outside": 18, "top": 22, "bottom": 25 },
    "first_indent": "1ch",
    "page_number_pos": "bottom_outside",
    "running_head": { "verso": "book_title", "recto": "chapter_title" }
  }
}
```

**요지**:
- 사용자 노출 필드는 `format.trim_id`, `format.theme_id`, `format.separator_id` **3개만**.
- 모든 디자인 토큰은 `_locked`에 들어가서 **렌더러 안에서만 산다**. JSON 표면적 0.
- 블록 타입 13종은 Atticus·Vellum 둘의 합집합(드롭캡·콜아웃 박스 포함). 1단계는 **`paragraph / scene_break / blockquote / image`** 4종만 구현해도 무방.

## 7. 섹션 D — 한국 시장 변수

| 한국 특수사항 | 위 권장이 그대로 유효? | 보완 |
|---|---|---|
| KLREQ 금칙 처리 | ✅ 유효 | Typst 한국어 패키지 또는 직접 금칙 테이블. 사용자 노출 0(자동). |
| 한영 혼식 자간 | ✅ 유효 (자동) | 사용자 노출 0. KLREQ가 자동 처리 권고. |
| 양끝 정렬 기본 | ✅ 유효 | 한국어 기본. 우리도 양끝(노출 안 함). |
| 한자 병기/루비 | ⚠️ 주의 | Vellum/Atticus 0%. **부크크/퍼플도 미지원.** 우리가 지원하면 차별화. JSON에 `[漢字]{ruby:한자}` 인라인 둘 것 권장(스키마 v2에 포함). |
| 세로쓰기 | 🚫 1단계 제외 | 시장 0%. 무시. |
| 부크크 비교 | 위 권장 유효 | 부크크는 조판을 안 함. 우리는 "조판해주는" 첫 한국 도구 → **부크크 사용자가 PDF 만들기 전 단계**를 흡수 가능. |
| 신국판 152×225 | 한국 표준 | 1단계 유일 판형으로 적절. (Atticus 17종/Vellum 17종은 inch 기반, 한국 신국판 없음 — 차별화 포인트.) |
| KoPub / 노토 | 노토 세리프 CJK KR이 안전 | KoPub도 무료지만 Hinting 이슈. 노토 권장 유지. |

**결론(한 줄)**: A/B/C 권장은 **한국에서도 그대로 유효**. 한 가지만 추가 — **한자 루비 인라인은 v2 스키마에 미리 자리 만들어 두자.** Vellum/Atticus가 못 하는 거고 부크크/퍼플도 못 한다. 한국어 단행본(특히 동아시아 고전·학술·역사) 시장의 진입 차별화.
