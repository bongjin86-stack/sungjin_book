# Reedsy Studio — UX 베이스라인 추출 (2026-05-20)

> **목적**: 무료 SaaS 중 가장 폴리시 높은 작가용 책 조판 도구의 패턴을 추출해
> 우리 `/editor`(typst.ts 기반)의 베이스라인으로 삼는다.
> 한국어/Typst에 맞게 변형할 부분은 "한국어 변형" 절에 명시.

## 0. 한 줄 결론

Reedsy = **"단순함 + 자동 조판 + 무료"의 표준**. 자유도 제한이 곧 품질 보장.
우리도 같은 철학(CLAUDE.md "자유도 = 품질 리스크")이라 패턴 그대로 이식 가능.

다만 **Reedsy는 실시간 미리보기가 약하다** — 우리(Typst SVG 실시간)가 오히려 앞섬.
즉 **Reedsy UX + Typst 미리보기 = 새 표준**.

**Cambric**(데스크탑·$199·Typst 엔진)이 우리와 거의 같은 기술 결로 영문권 POD 시장을 잡고 있음.
그쪽 디자인 옵션 10종 = **책 조판 도구의 산업 표준 체크리스트**. (15절 참고)

## 1. 전체 화면 구조

```
┌────────────────────────────────────────────────────────────────┐
│ 상단 바: 책 제목 · 자동저장 · [집필중] [검토] [완성] · 사용자  │
├──────────────┬──────────────────────────┬─────────────────────┤
│ 좌 사이드바  │      에디터 (중앙)        │   미리보기 (우)      │
│              │                          │                     │
│ ┌ 집필/포맷  │  ┌ 챕터 N 라벨           │   (실제 책 페이지)   │
│ │  탭        │  │ 챕터 제목 입력         │   SVG 또는 PDF      │
│ │           │  │                       │                     │
│ │ 목차       │  │  본문 입력 영역        │   페이지 N / M       │
│ │  [속표지]  │  │  (양끝정렬 자동)       │                     │
│ │  [목차]    │  │                       │                     │
│ │  [서문]    │  │  ─ 자동 들여쓰기 ─    │                     │
│ │  [제1장]   │  │                       │                     │
│ │  [제2장]   │  │                       │                     │
│ │  [저자소개]│  │                       │                     │
│ │           │  │                       │                     │
│ │ + 챕터추가 │  │                       │                     │
│ └           │  └ N자 · 맞춤법 · 저장    │                     │
└──────────────┴──────────────────────────┴─────────────────────┘
```

**3컬럼이 표준.** 좌(구조) · 중(쓰기) · 우(결과). 우리도 같은 구조.

## 2. 사이드바 (좌)

### 2-1 3구역 분리 (= 우리 matter blocks와 일치)
- **Front matter (앞부분)**: 헌사 · 에피그래프 · 서문 · 머리말 · 감사의 글 · 판권지(필수) · 목차(자동)
- **Body (본문)**: 챕터들
- **Back matter (뒷부분)**: 후기 · 저자소개 · 에필로그

각 구역에 **[편집]** 버튼 → 모달로 켤/끌 섹션 선택 (토글 스위치)

### 2-2 드래그 앤 드롭으로 챕터 순서 변경
**여섯 점(⠿) 아이콘 잡고 드래그.** 챕터 단위, matter 항목 단위 모두.

### 2-3 챕터 = 사이드바 아이템 1개
- 클릭 → 에디터/미리보기 둘 다 그 챕터로 점프
- 제목 직접 클릭해서 inline 편집
- 우리 현재 구조와 거의 동일

### 2-4 "+" 버튼 = 새 챕터 추가
사이드바 맨 아래.

---

## 3. 에디터 (중앙)

### 3-1 디스트랙션 프리
**광고·채팅·알림 0개.** 본문 + 최소 UI만.

### 3-2 스마트 포맷팅 바 (selection 시에만 등장)
**텍스트를 선택해야 나오는 부동(floating) 툴바.** 항시 노출 X.
- 볼드 / 이탤릭 / 헤딩 3단계 / 글머리표 / 번호 매기기 / 인용 / 정렬 / 링크 / 코멘트

### 3-3 자동 들여쓰기
**아무것도 안 해도 둘째 단락부터 1em 들여쓰기.** 우리 KLREQ I1과 일치.

### 3-4 우측 도구함 (보조 패널)
- **이미지 삽입** ("+" 버튼 또는 드래그)
- **장면 구분 (scene break)** — 가운데 별표 · 빈 줄 등 자동
- **각주/미주 삽입**
- **챕터 분할 (✂︎ 가위 아이콘)** — 점선 오버레이로 위치 지정 후 분할

### 3-5 챕터 메타 (상단 바)
"챕터 · 2장 · 직접 수정 가능 (프롤로그, 에필로그 등)" 같은 인라인 힌트.

### 3-6 하단 상태바
**N자** · **맞춤법 검사** · **저장 상태** (자동/수동 둘 다).

---

## 4. 미리보기 (우)

### 4-1 Reedsy의 약점
**실시간 미리보기는 약하다.** 본문 작성 화면은 디스트랙션 프리지 미리보기 없음.
미리보기는 **export 단계에서만** 작동 ("Real-time typesetting visualization" 마케팅 문구는 과장).

### 4-2 우리는 이미 앞섬
**Typst SVG가 입력 즉시(9~25ms) 갱신.** 챕터 클릭 자동 스크롤.
=> 이 부분은 Reedsy 따라가지 말고 **우리 강점 유지**.

---

## 5. 포맷팅 옵션 (디자인 자유도 제한)

### 5-1 폰트 자유도 = 거의 없음
- 글자 크기: **small / medium / large** (수치 X) ← KLREQ에서도 비슷한 방향
- 줄간격: 옵션 있음 (수치 X)
- 폰트 자체는 **테마가 결정.** 사용자 직접 못 고름.

### 5-2 테마 = 3개 고정
1. **Reedsy** — Lato + Merriweather (모던, 비소설)
2. **Classic** — Crimson (전통, 가장 책다움)
3. **Romance** — Crimson + 장식 챕터 제목

**테마가 모든 디자인 결정을 흡수.** 사용자는 글자만 쓴다.

### 5-3 챕터 옵션
- 챕터 번호 숨기기 (export 시 토글)
- 드롭캡 켜기/끄기 ✅
- 챕터 끝/책 끝 각주 위치

### 5-4 판형 (4종 고정)
- Trade 6 × 9″
- Digest 5.5 × 8.5″
- Reedsy 5 × 8″
- Pocket 4.25 × 6.87″

(우리는 신국판 152 × 225mm 1종 — 한국 규격)

### 5-5 출력
- **EPUB 3** (reflowable)
- **PDF/X-1a:2001** (인쇄 표준)

## 6. 워크플로 (7단계)

1. **원고 가져오기** — Word/Google Docs/타 서비스 임포트
2. **본문 포맷팅** — 단락·헤딩·인용·리스트
3. **이미지/장면 구분 삽입** — 우측 패널 Insert 버튼
4. **Front matter 구성** — 판권지(필수)·헌사·서문 등 토글
5. **표지 업로드** — 1:1.6, 최소 2500px
6. **판형 선택** — Exports 페이지
7. **출력** — 테마 선택 → Export book → PDF/EPUB

## 7. 협업 & 부가 기능

- **실시간 공동 작성** (Google Docs급)
- **변경 추적 + 코멘트**
- **베타 독자 미리보기** — 챕터 단위 공유 링크
- **온라인 e-reader** — 원고 검토용 뷰어
- **버전 히스토리** — 무료는 제한, 유료(Craft $4.99/mo)에 무제한
- **모바일 웹** — "홈 화면에 추가"로 사용. 전용 앱 없음.

| 플랜 | 가격 | 핵심 |

## 11. 즉시 적용할 패턴 (우선순위)

### A. 빠른 것 (1~2일, UX 임팩트 큼)
1. **챕터 드래그 앤 드롭** (⠿ 아이콘 + react-dnd 또는 dnd-kit)
2. **챕터 ✂︎ 분할** (커서 위치에서 분할)
3. **스마트 포맷팅 바** (selection 시 floating)
4. **자동저장 상태 명시화** (Vellum/Reedsy식: "방금 전 저장됨")

### B. 중간 (3~5일)
5. **드롭캡 옵션** (Typst `dropcap` 패키지)
6. **장면 구분 (scene break)** — 우리 matter 블록 추가
7. **이미지 삽입 UI** (Base64 또는 업로드)
8. **챕터 메타 라벨** ("프롤로그, 에필로그 등 직접 수정 가능")

### C. 느린 것 (1~2주, 출시 전 필수 아님)
9. Word/HWP 임포트
10. 표지 업로드 UI
11. 베타 공유 링크
12. 버전 히스토리

## 15. Cambric — Typst 동지 (참고급 자료)

### 15-1 기본 정보
- **회사/제품**: Cambric (cambric.pub) — 데스크탑 앱
- **가격**: $199 1회 결제, 30일 환불
- **플랫폼**: Windows + Mac (네이티브, 오프라인)
- **엔진**: **Typst** (우리랑 같음. 공개적으로 명시)
- **타깃**: 영문권 인디 출판 (KDP, IngramSpark 등)
- **CJK/한국어**: **언급 0건**. 영문 전용.

### 15-2 UI 구조
```
좌: 매뉴스크립트 바인더 (Parts → Chapters → 단어수, 드래그 앤 드롭)
중: 에디터 (디스트랙션 프리 + 스프린트 타이머)
우: 라이브 미리보기 (실제 PDF 렌더링, 페이지 단위)
```
→ **우리와 동일한 3컬럼.** Cambric 존재 자체가 우리 방향 확증.

### 15-3 디자인 옵션 10종 (= 산업 표준 체크리스트)

| # | 옵션 | 우리 현재 | 비고 |
|---|------|----------|------|
| 1 | Drop Caps (드롭캡) | ❌ | Typst dropcap 패키지로 추가 가능 |
| 2 | Scene Breaks (장면 구분) | ◐ (matter만) | 본문 내 ornament 미지원 |
| 3 | Running Headers (러닝 헤더) | ❌ | 짝수=책제목, 홀수=챕터제목 미러 |
| 4 | Custom Fonts (폰트) | ✅ | bodyFont 옵션 있음 |
| 5 | Margins & Trim (여백/판형) | ✅ (고정) | 신국판 1종, 사용자 노출 X (D-004) |
| 6 | Widows & Orphans | ◐ | Typst 본진 기본 동작에 일임 |
| 7 | Chapter Titles | ✅ | 챕터번호+제목 |
| 8 | Line Spacing | ✅ | lineSpacing 옵션 (narrow/normal/wide) |
| 9 | Page Numbers | ✅ | pageNumberPosition 옵션 |
| 10 | Paragraph Indent | ✅ | paragraphIndent 옵션 |

**갭 3개**: 드롭캡 · 장면 구분(본문 내) · 러닝 헤더.

### 15-4 부가 기능 (출시 후 검토)
- 9항목 사전점검(preflight) — 부크크 입고 전 체크리스트로 차용 가능
- 5개 리테일러 검증 (Amazon/Apple/Kobo/B&N/Google) — 우리는 부크크/교보가 대응
- 메타데이터 관리 (ISBN, 키워드, 카테고리)
- 로열티 계산기
- 리스팅 시트 자동 생성

### 15-5 Cambric에서 가져올 영감 (우선순위)

**즉시 박을 것**:
- ✅ **드롭캡** — Cambric/Vellum/Reedsy 다 있음. 챕터 첫 글자에 큰 한글자.
- ✅ **러닝 헤더** — 책 한 권 분량 단행본의 표시. 짝수/홀수 미러.
- ✅ **장면 구분 ornament** — `* * *` 또는 `⁂` 가운데 정렬.

**미루기**:
