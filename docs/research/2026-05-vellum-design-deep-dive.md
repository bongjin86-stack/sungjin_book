## 0. 한 줄 요약

Vellum의 비결은 "**책의 사진을 페이지의 주인공으로 만든다**"이다.
배경·UI·타이포는 전부 무채색 회색으로 빠지고, 컬러는 두 군데만 — (1) **자주색(#5d2e51 부근) 꽃 로고**, (2) 실제 책 표지/내지 미리보기 자체. 결과적으로 모든 페이지가 "도구의 화면이 아니라 책의 견본실"처럼 보인다.

## 1. 랜딩 페이지 비주얼 언어 (`vellum.pub`)

### 1.1 색·배경

| 영역 | 색 추정 | 비고 |
|---|---|---|
| 페이지 배경 | `#f4f4f3` 정도의 **콜드 그레이** (완전 흰색 아님) | 종이 톤 아닌 **회색 — "전시장 벽"** |
| 섹션 분리 | `#e8e8e8` / `#f4f4f3` 교차 | 그라데이션 없음. 박스도 없음. 색 한 단계만 바꿈 |
| 본문 텍스트 | `#222` 근처 다크 그레이 | 순흑색 회피 |
| 보조 텍스트·캡션 | `#777` | |
| 액센트 | **자주색 `#5d2e51`** (꽃 로고 + 헤딩 일부) | 시각 무게 5% 이하 |
| CTA 마젠타 핑크 | `#b056a6` 부근 | "WHAT WILL YOU CREATE?" 옆 동그란 챗 버튼 |

**핵심 결정**: 흰색이 아닌 **차가운 회색 배경 + 무채색 표면**. 책 미리보기 사진을 띄우면 *책이 따뜻해 보인다*. 콘트라스트 트릭이다.

### 1.2 히어로

- 최상단에 **MacBook Pro 목업 1장**. 노트북 안에 Vellum 앱 화면 + iPad/iPhone 미리보기가 들어 있다.
- 그 아래 작은 자주색 꽃 로고와 `VELLUM` 워드마크, 그리고 한 줄 카피 — `CREATE BEAUTIFUL BOOKS`
- 더 작은 글씨로 `Only Available for Mac`
- **CTA 버튼이 없다.** 다운로드는 더 아래에 작은 텍스트 링크.

→ 우리(SaaS) 본능과 반대다. "지금 가입하세요" 없이 **"보세요 → 갖고 싶어지세요"**.

### 1.3 섹션 리듬 (5박자)

```
HERO        — MacBook 목업 (회색 배경)
EBOOKS      — iPad + iPhone 미리보기 (왼쪽 텍스트, 오른쪽 디바이스)
PAINLESS    — 펼친 인쇄책 사진 (왼쪽 디바이스, 오른쪽 텍스트)
EFFORTLESS  — Kindle + 펼친 책 (양쪽 디바이스)
DESIGNED    — 앱 UI 그대로 (Build/Style/Preview/Generate 탭별 슬라이드쇼)
SOCIAL      — 작가 사진/표지/추천사 그리드 ("WHAT WILL YOU CREATE?")
PRICING     — 한 줄 표 ($249.99 / $199.99)
```

**규칙**: 헤딩은 **두운(alliteration)** — `Ebooks Everywhere`, `Painless Print`, `Effortless Style`, `Designed for Books`. 영어 카피 트릭이지만 *각 섹션이 한 단어 두 개로 묶인다*는 리듬 자체는 한국어로도 가져올 수 있다 ("**고운 글, 곧은 책**" 식).

### 1.4 타이포그래피

- **헤딩**: 산세리프, 자간 약간 늘림 (tracking ~50/1000), 전부 대문자. `DESIGNED FOR BOOKS` 같은 식.
- **본문**: 산세리프 14~16px, line-height 1.6 정도. 폰트는 시스템 산세리프(SF Pro/Helvetica Neue 추정).
- **본문에 세리프를 안 쓴다**가 핵심. 세리프는 **결과물 안에만** 살고 UI는 산세리프. 도구와 결과물의 시각적 역할이 분리됨.
- 한 페이지에 폰트 굵기 단계는 **2개만** (Regular + Bold).

### 1.5 디바이스 목업 디테일

캡처 `80-tab-01-build-s1.png`, `01-home.png`에서 확인:

- **그림자**: 매우 약함. `box-shadow: 0 8px 24px rgba(0,0,0,0.06)` 수준.
- **각도**: 정면 0도. 사선/3D 회전 없음.
- **베젤**: 실물 디바이스 정확. iPad는 베젤 두께·홈버튼 시대 모델까지 정확.
- **내부 컨텐츠**: 실제 Vellum 출력물의 스크린샷. 모형 텍스트(lorem ipsum) 절대 없음.

→ "**진짜 책을 진짜 디바이스에서 보여준다**"는 메시지. 합성티 안 나게 만드는 데 공을 들였다.

## 2. 앱 UI 디자인 (Mac 앱 스크린샷 분석)

기준: `24-text-editor.png`, `26-preview.png`, `25-styles.png`, `80-tab-*` 시리즈.

### 2.1 3분할 레이아웃의 색

| 패널 | 배경색 추정 | 역할 |
|---|---|---|
| 좌측 Navigator | `#f7f6f4` (살짝 따뜻한 라이트) | 챕터 트리 — 가장 차분 |
| 중앙 Editor | `#ffffff` | 본문 — 글이 주인공 |
| 우측 Preview | `#e8e6e3` (살짝 더 진한 회색) | 디바이스 배경 — "전시 케이스" 느낌 |

**핵심**: 3패널이 다 흰색이 아니다. **3단계의 회색 진폭** (왼쪽 따뜻 → 가운데 흰 → 오른쪽 차가운/진한). 이 그라데이션 자체가 "**왼쪽은 구조, 가운데는 글, 오른쪽은 결과**"라는 위계를 무의식적으로 전달한다.

### 2.2 패널 구분선

- **선이 없다.** 색 차이만으로 분리. 1px 라인 보더 절대 안 그림.
- 상단 툴바도 같은 회색 톤 — 패널이 *공기처럼* 합쳐짐.

→ 우리 트랙 B 에디터 `EditorThreePane.tsx`에 직접 적용 가능. CSS `border-right: 1px solid #ddd` 같은 거 빼고 배경색 단계차로 처리.

### 2.3 미리보기 디바이스 프레임

`26-preview.png` 기준:
- Kindle 미리보기에 **실제 Kindle 베젤** (둥근 모서리, 베젤 두께)이 들어가 있다.
- 디바이스 위·아래에 **드롭다운 1개씩** (위: 디바이스 선택, 아래: Draft/Proof 모드)
- 프레임 안 콘텐츠는 **실시간 본문**. 책 시뮬레이션이 아니라 *진짜 그 페이지*.

→ 우리 미리보기 패널에 **PDF 단일 페이지가 아니라 "스프레드 + 책 프레임"** 으로 감싸면 같은 효과. 우선순위는 낮지만 마무리 단계에 추가 가치.

### 2.4 사이드바 챕터 리스트의 시각 위계

`24-text-editor.png` 좌측:
- 가장 위 **책 제목** (Bold, 본문보다 1단계 큼)
- 그 아래 **Edited** 상태 라벨 (회색 작게)
- 챕터들: 들여쓰기 없음, **선택된 챕터만 옅은 파란 배경** (`#d8e7f3` 정도)
- Front matter / Body / Back matter 그룹 라벨은 **얇은 구분선 없이** 작은 캡션으로만 표시

→ 우리 챕터 리스트 컴포넌트도 *시각 무게가 거의 없도록* 만들 것. "들여쓰기/아이콘/뱃지" 본능적으로 넣지 말 것.

### 2.5 식자법 — 챕터 시작 페이지

`26-preview.png`, `80-tab-03-preview-s2.png`에서 챕터 첫 페이지 식자 디테일:

| 요소 | 디테일 |
|---|---|
| 챕터 번호 | 작게(본문보다 약간 작음), 양각 — `CHAPTER 1` 식 |
| 챕터 제목 | 그 아래 큰 글씨, 본문 폰트의 1.5~1.8배 |
| 드롭캡 | 첫 문단 첫 글자가 본문 3~4줄 높이로 떨어짐. 본문 폰트와 다른 폰트 (대문자 세리프) |
| 위 여백 | **본문 영역의 위 1/3을 비움** — "쉼표" |
| 쪽번호 | 챕터 시작 페이지엔 없음 |

→ 우리 신국판 Classic 사양과 **거의 동일**. 한국어판으로 옮길 때 드롭캡 대신 **챕터 번호 큰 글자 1자** (예: "一" "二") 로 대체하면 한국어 무게감이 산다.

### 2.6 "도구이면서도 예쁜 결과 나올 것 같은" 마이크로 디테일

- 툴바 아이콘은 **모두 단색 라인 아이콘**. 컬러 없음. 둥근 모서리 2px.
- 상단 탭(`Build / Style / Preview / Generate / Update`) — 활성 탭은 **검정 텍스트**, 비활성은 회색. 밑줄·배경 없음.
- 폰트 크기 슬라이더가 있는데 **숫자 값을 표시 안 한다** (`50-styles-body.png`). "12pt"가 아니라 *슬라이더 위치*. **숫자를 노출하지 않는 게 안심을 준다**.
- 스타일 카드(`25-styles.png`): 카드마다 **미니어처 펼친 책 사진** 들어감. 카드 자체가 "이 스타일을 고르면 이렇게 나온다"의 견본.

→ **우리 D-011 (스타일 카드 = 견본 미니어처)** 방향과 정확히 일치. 1단계 Classic 카드 하나여도 미니어처는 정성껏.

## 3. 갤러리/사례 페이지 (`created.vellum.pub`)

`04-gallery.png` 분석:

### 3.1 레이아웃

- **2열 그리드**가 아닌 **세로 1열 카드 스택** (큰 화면에서도).
- 카드 1개 = [표지 큰 이미지 좌측] + [제목·저자·시놉시스·추천사·구매링크 우측].
- 카드 사이 간격 약 80px. 한 화면에 1.5~2개만 보임.

### 3.2 카드 비주얼

| 요소 | 디테일 |
|---|---|
| 책 표지 | 폭 ~280px. **드롭섀도 약간** (옅은 회색 1px 외곽 + 8px blur). 둥근 모서리 없음 — **각진 책 그대로**. |
| 제목 | 산세리프 Bold, ~22px |
| 저자 | "by [Author]" — 본문 사이즈, 회색 |
| 시놉시스 | 본문 사이즈, 4~6줄 |
| 추천사 | **이탤릭 + 인용부호**, *"The only bad thing about Vellum is..."* — **저자 본인의 말**이 SNS 인용처럼 들어감 |
| 구매 링크 | 텍스트 링크 가로 나열 (Amazon · Apple · B&N · Kobo). 버튼 아님. |

### 3.3 왜 효과적인가

- **표지를 작게 모은 갤러리가 아니다.** 책 1권에 1화면 가까이 줘서 "이건 책이지 카탈로그가 아니다"를 강조.
- 일부 카드는 **표지 자리만 비어 있음** (`04-gallery.png` 하단 4개). "여러분도 여기 들어갈 수 있다"는 빈자리.
- 추천사가 **출판사 광고**가 아니라 **저자가 Vellum에 대해 한 말** — 사회적 증명이 자연스럽다.

→ 우리 트랙 C에서 `book.sungjin.kr/created`를 만들 때 **세로 1열, 표지 크게, 저자 인용** 구성이 정답. 그리드는 함정.

## 4. 헬프/튜토리얼 사이트의 톤 (`help.vellum.pub`)

`23-text-features.png` (캡처 원본) 기준 — 이 캡처가 헬프 사이트의 시각 언어를 가장 잘 보여준다.

### 4.1 비주얼

| 요소 | 디테일 |
|---|---|
| 헤더 | 좌측에 **작은 자주색 꽃 로고** + `Vellum Help`. 그 외 메뉴 없음. 우측에 검색 |
| 사이드바 | **우측**(왼쪽이 아니라). 현재 페이지 안 목차만. 자주색 텍스트. |
| 본문 | 산세리프, 짙은 그레이 텍스트, 흰 배경. 폭 ~720px로 좁힘. |
| 푸터 | `VELLUM   CREATE BEAUTIFUL BOOKS` — 회색 띠. 매 페이지에 같은 슬로건 반복 |

### 4.2 스크린샷 처리 — **이게 가장 중요한 발견**

- **화살표·하이라이트 박스·콜아웃 거의 없다.**
- 대신 **스크린샷 자체를 크롭해서 필요한 부분만 보여준다.** 가령 "메뉴를 여는 법"을 설명할 때 전체 앱이 아니라 *그 메뉴가 펼쳐진 5cm × 8cm 부분*만 잘라 붙임.
- 스크린샷 외곽에 **얇은 그림자 1px만** — 둥근 모서리도 거의 없음.

→ **레슨**: "화살표로 가리키기" 대신 "필요한 영역만 잘라 보여주기"가 정보 밀도 높다. 우리 도움말 만들 때도 *전체 에디터 스크린샷에 빨간 동그라미 그리기* 본능을 거를 것.

### 4.3 카피 톤 — 안심 패턴 수집

캡처 분석과 본문 fetch에서 수집한 *직접 인용*:

- > "Clicking on a style will select it, applying the style immediately to your entire book."
- > "Vellum will open the Word document, analyze it to find title information and chapters, and then convert it into Vellum's native format."
- > "You can experiment with styles as you are writing, or come back and change your mind after you've proofed your book."
- > "More Control, Less Guesswork" (4.1 버전 슬로건)
- > "Choose your trim size, and Vellum does the rest."
- > 두운 카피: "Painless Print", "Effortless Style", "Ebooks Everywhere"

**공통 구조**: `[사용자 동작 1개] → [Vellum이 알아서 처리하는 것 N개]`
한국어로 옮기면:
- "**판형 하나만 고르세요. 나머지는 자동입니다.**"
- "**스타일을 클릭하면 책 전체에 즉시 적용됩니다.**"
- "**글은 워드에서, 책은 sungjin에서.**"

## 6. Atticus와의 차이 (짧게)

- Atticus는 **웹앱**이지만 UI에 **컬러 박스/배경 그라데이션/3D 책 회전**이 많아 *SaaS*처럼 보임.
- Vellum은 *데스크탑 앱*이라 더 정적이지만, 그 정적임이 곧 **"진짜 출판 도구"** 이미지로 환산됨.
- 우리는 **웹앱**이라는 Atticus 쪽이지만, **시각 언어는 Vellum**을 택해야 한다. *조합이 차별화*.
