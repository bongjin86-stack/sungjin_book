# 현재 작업 상태

> 마지막 업데이트: 2026-05-19 KST
> 단계: 트랙 B MVP v1 — 고객 흐름을 "책 생성 → 구조 자동 생성 → 블록별 편집 → 자동 목차 → 미리보기"로 재정렬 중.

## 지금 위치

- 웹 에디터는 Next.js 14 + TypeScript + Tailwind + BlockNote 기반.
- 시작 가이드에서 책 기본 정보, 구성, 테마를 고른 뒤 `/editor`로 진입한다.
- 새 책은 `속표지 / 판권지 / 목차 / 1장 / 저자 소개` 흐름을 기본으로 가진다.
- 사이드바는 `앞부분 / 본문 / 뒷부분`으로 묶인다.
- 챕터는 부제목, 목차 표시 여부, 목차용 이름, 챕터 번호 표시 여부를 저장한다.
- 목차는 직접 입력하는 페이지가 아니라 챕터 목록에서 자동 계산한다.
- 속표지, 판권지, 목차, 일반 앞/뒤 구성 블록은 챕터 폼 대신 전용 편집 화면으로 열린다.
- 기존 localStorage 데이터는 읽을 때 새 구조 기본값을 보정한다.

## 이번 세션에서 한 일

- `copyright` 블록 타입 추가.
- `ChapterBlock`에 `subtitle`, `includeInToc`, `tocTitle`, `showChapterNumber` 추가.
- `BlockEditor` 추가: 속표지, 판권지, 목차, 일반 구성 블록 편집 UI 분리.
- `ChapterForm` 확장: 부제목, 목차 표시, 목차용 이름, 챕터 번호 표시 입력 추가.
- `PreviewPanel` 확장: 부제목과 챕터별 번호 표시 여부 반영.
- `Sidebar` 정리: 앞부분/본문/뒷부분 그룹 표시.
- 챕터 화면에서 목차 설정을 제거하고, 챕터 번호 표시는 접힌 `챕터 설정`으로 이동했다.
- 미리보기 하단 문구를 `Page`에서 `이전 쪽 / 다음 쪽`, `1 / N쪽`으로 바꿨다.
- 검증: `npm.cmd run build` 통과.
- 공식 기준 문서 추가: `docs/research/atticus-vellum-workflow-official.md`

## 다음 작업 후보

1. 미리보기 조판 보정: 긴 문단 줄 단위 분할, 실제 글자폭 기반 줄바꿈, 홀짝 쪽번호.
2. 저장 후 다음 챕터 작성 흐름: 원고를 계속 붙여넣는 고객을 위해 다음 챕터 입력 상태로 자연스럽게 이동.
3. 전체 책 미리보기: 속표지부터 저자 소개까지 한 흐름으로 넘겨보기.
4. 목차 페이지 번호 연결: 페이지 계산 안정화 이후 목차에 실제 쪽번호 표시.
5. 판권지 필드 고도화: 저작권, 발행일, ISBN, 출판사 입력을 구조화.
