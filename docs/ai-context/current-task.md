# 현재 작업 상태

> 마지막 업데이트: 2026-05-19 KST
> 단계: 트랙 B MVP v1 — Atticus/Vellum 흐름을 참고한 책 집필/미리보기 개선 중

## 지금 위치

- 웹 에디터는 Next.js 14 + TypeScript + Tailwind + BlockNote 기반이다.
- 시작 가이드에서 기본 정보, 책 구성, 테마를 고른 뒤 `/editor`로 진입한다.
- 새 책은 `속표지 / 판권지 / 목차 / 제1장 / 저자 소개` 흐름을 기본으로 가진다.
- 목차는 사용자가 직접 쓰는 페이지가 아니라 챕터 목록에서 자동 계산하는 시스템 블록이다.
- 챕터 입력 화면은 제목과 본문 붙여넣기 중심으로 단순화했다.
- 챕터 번호, 목차 포함, 챕터 번호 표시 같은 세부 조판 옵션은 고객 첫 화면에서 숨겼다.
- 새 챕터 작성 버튼은 `저장하고 다음 챕터` 흐름으로 바꿨다.
- 사이드바의 `간지 추가` 버튼은 아직 개념이 덜 잡혀 있어 임시 제거했다.
- 미리보기 페이지 분할은 긴 문단을 줄 단위로 나눠 다음 쪽으로 넘긴다.

## 이번 세션에서 한 일

- `ChapterForm`에서 챕터 번호 직접 입력, 부제목 입력, 챕터 설정 접힘 영역, 준비 중 맞춤법 버튼을 제거했다.
- 새 챕터 저장 버튼 문구를 `저장하고 다음 챕터`로 바꿨다.
- 새 챕터 번호 표기를 `제N장` 형식으로 맞췄다.
- 사이드바에서 `간지 추가` 버튼을 제거하고 챕터 수 표시가 실제 챕터만 세도록 바꿨다.
- 검증: `npm.cmd run build` 통과.

## 남은 작업 번호표

1. 챕터 입력 화면 단순화 — 완료
2. 저장하고 다음 챕터 작성 흐름 — 완료
3. 집필/포맷팅 탭 위치 재설계
4. 전체 책 미리보기
5. 목차 실제 쪽번호 연결
6. 앞부분 로마 숫자/번호 규칙
7. 판권지 필드 구조화
8. 실제 글자폭 기반 줄바꿈
9. 홀짝 쪽번호와 바깥쪽 위치
10. PDF 출력 경로 연결

## 다음 작업 우선순위

1. 집필/포맷팅 탭 재설계: 고객이 “원고 입력”과 “책 전체 스타일 선택”을 헷갈리지 않게 나눈다.
2. 전체 책 미리보기: 속표지부터 저자 소개까지 실제 책 흐름으로 한 번에 넘겨본다.
3. 목차 쪽번호 연결: 전체 책 미리보기 계산이 안정된 뒤 실제 쪽번호를 목차에 넣는다.

## 주의

- 조판 핵심 경로에서 LLM API 호출 금지.
- 고객에게 폰트, 자간, 여백 같은 자유도를 과하게 노출하지 말 것.
- 챕터 기반 책 MVP를 먼저 제대로 만든다. 챕터 없는 책은 이후 검토한다.
- Atticus/Vellum 관련 판단은 `docs/research/atticus-vellum-workflow-official.md`를 우선 기준으로 삼는다.
