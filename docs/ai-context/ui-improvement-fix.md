# 성진북스 웹 UI 버그 수정 지시서 (Claude Code)

**대상**: Claude Code (실행 에이전트)

이 문서는 이전 커밋(`94f9896`)에서 구현된 미리보기 패널의 버그를 수정하기 위한 지시서입니다.

---

## 1. 현재 문제점 진단

1.  **여백(Padding) 미적용 버그**:
    *   `PreviewPanel.tsx`의 본문 영역 `div`에 `paddingTop: "10%"` 등 퍼센트(%) 기반 패딩이 적용되어 있습니다.
    *   **원인**: 부모 요소(`absolute inset-0`)가 명확한 높이/너비를 가지지 못하거나, 퍼센트 패딩이 예상대로 작동하지 않아 텍스트가 종이 끝에 딱 붙어서 렌더링되고 있습니다.
2.  **비율 및 크기 문제**:
    *   신국판 비율(`152 / 225`)은 맞으나, 패널 너비(`w-[450px]`)에 비해 종이 컨테이너(`w-[340px]`)가 너무 커서 스크롤바가 생기거나 답답해 보일 수 있습니다.
3.  **조판·고급 옵션 미반영**:
    *   사이드바에서 옵션을 변경해도 미리보기에 즉각 반영되지 않는 부분이 있습니다. (특히 쪽번호 위치 등)

---

## 2. 수정 지침 (PreviewPanel.tsx)

### 2.1. 여백(Padding)을 px 또는 rem 단위로 고정
퍼센트(%) 대신 명시적인 픽셀(px) 값으로 여백을 설정하여 안정적으로 렌더링되게 합니다. 신국판(152x225) 비율의 축소판이므로, 그에 비례하는 픽셀 값을 사용합니다.

*   수정 전: `paddingTop: "10%", paddingBottom: "12%", paddingLeft: "13%", paddingRight: "11%"`
*   **수정 후**: `padding: "32px 28px 36px 34px"` (위 32px, 오른쪽 28px, 아래 36px, 왼쪽 34px - 대략적인 비율)

### 2.2. 종이 컨테이너 크기 조정
패널 너비(`w-[450px]`) 안에서 여유롭게 보이도록 종이 너비를 살짝 줄입니다.

*   수정 전: `className="relative w-[340px]"`
*   **수정 후**: `className="relative w-[300px]"` (또는 `320px`)

### 2.3. 쪽번호 위치 (Absolute Positioning) 수정
쪽번호가 종이 영역 밖으로 나가지 않도록 `bottom`, `right` 등의 값을 px 단위로 명확히 줍니다.

*   `bottom-center`: `bottom: "15px", left: 0, right: 0, textAlign: "center"`
*   `bottom-outside`: `bottom: "15px", right: "28px"` (우측 여백과 동일하게)
*   `top-outside`: `top: "15px", right: "28px"`

### 2.4. 챕터 번호 렌더링 수정
현재 `1111111` 처럼 이상하게 나오는 챕터 번호를 수정합니다. `chapterNum` 변수에 들어오는 값이 무엇인지 확인하고, "제 1 장" 형태로 깔끔하게 렌더링되도록 합니다. (현재 정규식 `.replace(/장$/, "")` 부분이 의도치 않게 작동할 수 있습니다.)
