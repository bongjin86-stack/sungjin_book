## 2026-04-27 — 프로젝트 시작

### D-001. 1단계는 웹 없이 Typst R&D만

**결정**: 회원가입·결제·인쇄 연동·웹 UI 모두 1단계에서 제외.

**이유**:
- 핵심 가설은 "콘텐츠 JSON이 깨끗한 PDF로 컴파일되는가" 단 하나.
- 웹/DB 먼저 만들면 가설 검증이 늦어지고, 가설 실패 시 매몰비용만 커진다.
- PDF 품질이 안 나오면 그 위에 뭘 올려도 의미 없음.

**대안과 기각 사유**:
- 웹 + Typst 동시 진행 → 컴파일 성공 후 정렬·금칙 미세조정에 시간 다 쓸 가능성.
- LaTeX → 한글 처리 의존성 무겁고, 모던 워크플로 아님.
- IDML/InDesign 자동화 → 라이선스 + 파이프라인 복잡도 폭증.

### D-002. AI는 핵심 경로에 두지 않는다

**결정**: 조판 파이프라인에서 LLM API 호출 0회.

**이유**:
- 비결정성 = 같은 입력에 다른 출력 = 디버깅 지옥.
- 외부 서버에 원고 보내는 건 프라이버시 리스크.
- 비용/속도/예측가능성/프라이버시를 한 번에 해결.
- AI는 콘텐츠 작성 보조(나중에 사용자 측에서)에만 검토.

### D-003. 스타일 선택 → 콘텐츠 입력 순서

**결정**: 사용자 흐름은 (스타일 미리보기 → 선택 → 콘텐츠 입력).

**이유**:
- 빈 페이지 공포증. 사용자 동기는 "예쁜 디자인 보고 거기 내 글 넣고 싶다"에서 출발.
- 스타일이 1개면 의미 약하지만, 데이터 구조를 처음부터 이렇게 잡아야 확장 가능.

### D-004. 사용자에게 폰트/자간/여백 노출 금지

**결정**: 포맷팅은 단락·제목·강조·인용까지만.

**이유**:
- 자유도 ↑ = 품질 리스크 ↑. 비전문가가 자간 만지면 망친다.
- "디자이너 없이 단행본 만들기"가 본 서비스의 핵심 약속.
- 결과물 일관성으로 브랜드 만들기.
