# 성진북스 웹 UI 개선 지시서 v2 (Claude Code)

**작성일**: 2026-05-19  
**작성자**: Manus  
**대상**: Claude Code (실행 에이전트)

이 문서는 Vellum과 Atticus 등 글로벌 출판 소프트웨어의 UI 구조를 분석한 결과를 바탕으로, 성진북스 웹 에디터의 "미리보기 패널(PreviewPanel)"을 "진짜 책"처럼 보이도록 개선하는 작업 지시서입니다.

---

## 1. 배경 및 목표

현재의 미리보기 패널은 단순한 흰색 카드에 텍스트가 나열된 형태라 "책을 만들고 있다"는 느낌을 주지 못합니다. Vellum과 Atticus의 핵심 성공 요인은 **"미리보기를 실제 디바이스나 종이책 프레임 안에 가두어 보여주는 것"**입니다.

목표: 오른쪽 미리보기 패널을 단순한 컨테이너가 아닌, **"신국판 종이책 프레임"**으로 시각화하여 사용자 경험을 극대화합니다.

---

## 2. PreviewPanel.tsx 개선 지침

### 2.1. 레이아웃 및 크기 조정
*   **패널 너비 확대**: 현재 `w-[380px]`에서 `w-[450px]` 정도로 넓혀 책 한 페이지가 여유롭게 보이도록 합니다.
*   **배경색**: 패널 전체 배경은 어두운 회색(`bg-[#1A1A1A]`)을 유지하여 중앙의 흰 종이가 돋보이게 합니다.

### 2.2. "책 프레임" 시각화 (핵심)
단순한 `div`가 아니라, 책의 두께감과 질감을 표현하는 CSS를 적용합니다.

*   **종이 컨테이너**:
    *   비율 고정: `aspect-ratio: 152/225` (신국판 비율)
    *   크기: `w-[320px]` 또는 `w-[340px]` (패널 너비에 맞게 조정)
    *   배경: 순백색(`bg-white`) 또는 미세한 미색(`bg-[#FAFAFA]`)
*   **책 두께 및 그림자 효과 (Book Binding Effect)**:
    *   왼쪽 제본 영역(Spine) 표현: `border-l-[4px] border-l-[#E0E0E0]` 또는 선형 그라데이션(`bg-gradient-to-r from-[#EAEAEA] to-transparent w-4 absolute left-0 h-full`)을 사용하여 책이 접히는 안쪽 여백의 음영을 표현합니다.
    *   입체감 그림자: `shadow-[0_10px_30px_rgba(0,0,0,0.5)]` 등 깊이감 있는 그림자를 적용하여 종이가 떠 있는 느낌을 줍니다.

### 2.3. 타이포그래피 및 여백 (Typst 근사치)
*   **여백 (Margin)**: Typst의 안20/밖18/위22/아래25mm 비율을 CSS 패딩으로 근사하게 적용합니다. (예: `pt-[10%] pb-[12%] pl-[13%] pr-[11%]`)
*   **본문 폰트**: `font-serif` (Noto Serif KR 적용)
*   **본문 크기**: `text-[13px]` (10pt 근사)
*   **줄간격**: `leading-[1.8]` (18pt 근사)
*   **양끝 정렬**: `text-justify`
*   **들여쓰기**: `paragraphIndent` 옵션이 켜져 있을 경우, 첫 단락 제외하고 `text-indent: 1em` 적용.

### 2.4. 쪽번호 및 챕터 헤더
*   **챕터 헤더**: `showChapterNumber` 옵션에 따라 상단에 "제 N 장" 표시.
*   **쪽번호**: 하단에 절대 위치(`absolute bottom-[5%]`)로 배치. `pageNumberPosition` 옵션에 따라 좌/우/가운데 정렬.

## 3. Claude Code 작업 지시 (Task Execution)

Claude Code는 다음 순서로 작업을 수행하십시오.

1.  **`web/components/editor/PreviewPanel.tsx` 수정**: 2절의 지침에 따라 "책 프레임" 디자인을 적용합니다. 단순한 흰색 박스가 아니라, 제본 영역의 음영과 입체적인 그림자가 들어간 실제 책 페이지처럼 보이도록 CSS를 정교하게 작성하십시오.
2.  **로컬 테스트**: `npm run dev` (또는 `pnpm dev`)를 실행하여 브라우저에서 미리보기 패널이 실제 책처럼 보이는지, 사이드바 옵션 변경 시 실시간으로 반응하는지 확인합니다.
3.  **GitHub Push**: 검증이 완료되면 변경 사항을 커밋하고 `main` 브랜치에 푸시합니다. 커밋 메시지는 `feat: 미리보기 패널 책 프레임 디자인 적용 (Vellum 스타일)`로 작성하십시오.

**주의사항**:
*   이 작업의 핵심은 **"시각적 프레임(Visual Frame)"**입니다. 사용자가 미리보기를 볼 때 "아, 내 책이 이렇게 인쇄되겠구나"라고 직관적으로 느낄 수 있도록 CSS 디테일(그림자, 제본선 음영)에 신경 써 주십시오.
*   모든 실행(Execution)은 Claude Code가 담당합니다.
