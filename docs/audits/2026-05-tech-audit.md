## P0 — 트랙 A 자체 결함

지금 PDF는 "PDF로 컴파일은 되지만, 인쇄소·국립중앙도서관에 들고 가면 거절당하는 상태"다.

### A1. 블리드(bleed) 0mm

- **현재 상태**: `template.typ:50-52` — 페이지 폭/높이가 트림(152×225) 그대로. 블리드 영역 없음.
- **영향**: POD 인쇄 시 외곽에 닿는 글자/도판이 잘림. 인쇄소가 "재단 여유 3mm 추가해주세요" 반려.
- **권장 조치**: 페이지를 158×231mm로 확장 + 안쪽 여백을 +3mm 보정. 트림 마크는 후처리.
- **작업량**: S
- **트리거**: 트랙 C 인쇄 연결 전. 트랙 B에서도 미리보기에서 "인쇄용 PDF" 옵션을 노출하려면 필요.

### A2. 프론트매터·백매터 없음

- **현재 상태**: `template.typ` 전체 — 표제지(half-title) / 판권면(copyright) / 차례(toc) / 헌사(dedication) / 머리말(preface) / 저자 후기(afterword) / 색인(index) 모두 없음.
- **영향**: ISBN 등록 시 "도서 형식 미충족" 반려. 국립중앙도서관 납본도 막힘.
- **권장 조치**: JSON 스키마에 `frontmatter`/`backmatter` 블록 추가. 템플릿에 별도 함수.
- **작업량**: M
- **트리거**: 트랙 C 시작 전. 트랙 B 스키마 v2에 함께 들어가야 함 (B3 참조).

### A3. 인용문 `emph()` — 합성 italic

- **현재 상태**: `template.typ:101-104` — `quote` 블록을 `emph(blk.text)`로 처리.
- **영향**: 노토 세리프 KR에는 italic 자형이 없어서 시스템이 강제 기울임(synthetic italic) 적용. KLREQ 한국어 조판 가이드 어긋남.
- **권장 조치**: `emph` 제거. 인용 표시는 들여쓰기 + 좌우 인용 부호 + 출처 우측 정렬 같은 시각 신호로.
- **작업량**: S
- **트리거**: 트랙 A 다음 컷.

### A4. quote block 위/아래 여백 0

- **현재 상태**: `template.typ:101` — `block(inset: (left: 2em, right: 2em))`만 있음. `top`/`bottom` 없음.
- **영향**: 본문과 시각적 분리가 안 돼서 인용인지 본문인지 구분 어려움.
- **권장 조치**: `inset`에 top/bottom 0.5em 또는 그 위에 `v(0.5em)` 추가.
- **작업량**: S
- **트리거**: A3과 함께.

### A5. 첫 챕터 `pagebreak(to: "odd")`로 곧장 본문

- **현재 상태**: `template.typ:25` — chapter 함수가 무조건 홀수 페이지로 점프. 첫 페이지가 표제지여야 하는데 곧장 1장 본문.
- **영향**: 책 구조 어긋남. "1쪽이 표제지" 통례 위반.
- **권장 조치**: A2 프론트매터 도입 시 자연 해결. 임시로는 `book()` 시작에 빈 표제지/판권면 페이지 삽입.
- **작업량**: S (A2와 함께)
- **트리거**: A2와 동시.

### A6. 쪽번호가 처음부터 아라비아

- **현재 상태**: `template.typ:54-65` — counter(page) 그대로 사용.
- **영향**: 출판 관행상 프론트매터는 로마자(i, ii, iii), 본문 첫 챕터부터 아라비아 1로 리셋. 안 그러면 "책 같지 않다"는 인상.
- **권장 조치**: `counter(page).update(1)` + numbering 전환을 본문 시작 시 호출.
- **작업량**: S
- **트리거**: A2와 동시.

### A7. 러닝 헤드(running head) 없음

- **현재 상태**: `template.typ` 전체 — header 미설정.
- **영향**: 200쪽 넘는 책에서 독자가 "내가 몇 장 어디쯤" 못 잡음. 단편집/논픽션은 거의 필수.
- **권장 조치**: 좌 페이지 = 책 제목, 우 페이지 = 챕터 제목. set page header에 context로.
- **작업량**: M
- **트리거**: 트랙 A 다음 컷. 200쪽 미만 시집 등은 끌 수 있게 옵션화.

### A8. widow/orphan 제어 없음

- **현재 상태**: `template.typ:82-86` — set par에 widow/orphan 관련 설정 없음.
- **영향**: 챕터 끝에 한 줄만 남거나 첫 줄이 페이지 하단에 떨어짐. 보기 흉함.
- **권장 조치**: Typst 0.14의 `par(linebreaks: ...)` 또는 챕터 내부에서 `block(breakable: ...)` 활용. (Typst의 widow 제어는 아직 제한적이므로 트랙 B 실데이터로 검증 후 결정.)
- **작업량**: M
- **트리거**: 트랙 A 다음 컷.

### A9. CMYK·색 프로파일 미지정

- **현재 상태**: `template.typ` — 색 지정 없음. Typst 기본은 RGB.
- **영향**: 인쇄소에서 RGB → CMYK 자동 변환 시 색이 어둡거나 채도가 빠짐. 표지·이미지 도판 들어가면 즉시 클레임.
- **권장 조치**: Typst 자체는 PDF 색공간 강제 지원이 약함. Ghostscript/pdfcpu 후처리로 PDF/X-1a 변환 단계를 트랙 C에 둔다.
- **작업량**: M (후처리 파이프라인 신설)
- **트리거**: 트랙 C 인쇄 연결 전.

### A10. 면수 4의 배수 보장 X

- **현재 상태**: `template.typ` — 마지막 페이지 처리 없음.
- **영향**: POD 무선제본은 4쪽(시그니처에 따라 16쪽) 단위로 제본. 안 맞으면 인쇄소가 빈 페이지를 임의로 추가하거나 반려.
- **권장 조치**: book() 마지막에 현재 페이지 카운터 보고 빈 페이지 자동 패딩.
- **작업량**: S
- **트리거**: 트랙 C 인쇄 연결 전.

### A11. 체크리스트 미완 검증 항목

- **현재 상태**: `docs/ai-context/checklist.md:30-31, 41-45` — 금칙처리, 한영 자간, 양끝 정렬, 들여쓰기, 챕터 분기, 쪽번호 위치 모두 사용자 육안 점검 미수행.
- **영향**: 트랙 A "첫 컷" 통과 미확정. 트랙 B에서 같은 템플릿을 재사용하기 전에 결함이 묻힐 수 있음.

## P2 — 트랙 C — ISBN/POD/커머스

### C1. 출판사 등록·책임

- **함정**: "전 사용자의 책을 우리 출판사 imprint로 발행" = 명예훼손·저작권 침해 콘텐츠도 우리가 1차 책임.
- **선택지**:
  - (a) BYO 출판사 코드 (사용자가 자기 출판사 등록)
  - (b) 우리 imprint + 자동 검수 게이트
  - (c) 우리 imprint + 약관 면책 (법적 효력은 약함)
- **작업량**: L (법무 자문 포함)
- **트리거**: 트랙 C 1주차.

### C2. 납본 의무

- **현재 상태**: 국립중앙도서관 + 국회도서관 2부씩 = 종이책 4부 비용 + 발송 자동화. 전자책은 파일 등록.
- **권장 조치**: POD 자동 발주 워크플로우에 "납본 4부" 자동 추가.
- **작업량**: M
- **트리거**: 트랙 C 종이책 출시 시.

### C3. ISBN 사후 변경

- **함정**: ISBN 박힌 후 콘텐츠 수정 = 다른 책으로 봐야 함.
- **권장 조치**: "수정판"인지 "오탈자(같은 ISBN)"인지 정책 + UI 게이트.
- **작업량**: M
- **트리거**: 트랙 C 사용자 노출 전.

### C4. CMYK·블리드·트림 마크

- A1·A9 연장. Typst → Ghostscript/`pdfcpu` 후처리.
- **작업량**: M
- **트리거**: 트랙 C 인쇄 연결 전.

### C5. 책등(spine) 두께 자동 계산

- **함정**: 면수 × 종이두께. 표지 디자인 입력 UI 필요 (현재 표지 0%).
- **작업량**: L (표지 디자이너 트랙과 동시).
- **트리거**: 트랙 C 종이책 출시 시.

### C6. 표지 디자이너

- 부크크·퍼플은 템플릿 + 사용자 이미지. 별도 트랙. 본 감사 범위 외.
- **트리거**: 트랙 C 시작 후.

### C7. 유통 (교보·예스24·알라딘)

- 각자 메타데이터 포맷·ingest 스펙 다름. ONIX 표준 권장.
- **작업량**: L (3사 각각 협의).
- **트리거**: 트랙 C 안정화 후.

### C8. 저자 인세 정산

## P2 — 부크크·퍼플 대비 차별화 위험

### D1. "ISBN까지 무료"의 단가

- **함정**: 2026 기준 ISBN 자체는 KOLIS-NET 무료지만 출판사 등록·납본·관리에 사람 손 들어감.

- **작업량**: S (스프레드시트).

- **권장 조치**: C1·C10 검수가 받쳐야 무료가 가능. 검수 우선 트리거.
- **트리거**: C1과 동시.
